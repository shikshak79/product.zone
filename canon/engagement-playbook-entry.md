# Product.Zone — Engagement Playbook: Entry Product
## Positioning & Messaging Sprint
**Canonical Reference · v1.1 · March 2026**
*Aligned to Buying Path X-RAY Architecture v5.1*

---

## Playbook Philosophy

**Sessions are decision-making moments, not working sessions.**

All analytical and preparatory work happens before the session. The founder arrives to find the analysis done, the options visible, and a recommendation already on the table. Their job is to choose, react, and validate — not to produce.

This means:
- Founders never feel burdened by homework they didn't do
- Sessions move fast because context is already built
- Quality of decisions is higher because options are pre-structured
- The engagement scales — 80% of prep work is systematically automatable

**Implicit homework over explicit tasks.**
Rather than assigning tasks ("go test this message with two clients"), the playbook creates conditions for observation ("how did the new framing land in conversations this week?"). Low hurdle. No compliance pressure. High signal when founders engage naturally.

## Pre-Engagement: Intake

Before the kickoff call, the founder completes intake. This is the foundation for all preparatory work.

### Intake Checklist

| Material | Required | Notes |
|---|---|---|
| Intake questionnaire | Required | Fillout.com form — covers stage, ICP hypothesis, current challenges, recent wins/losses |
| Homepage URL | Required | Live URL, not screenshot |
| Sales deck | Required | PDF or shareable link |
| LinkedIn profile | Required | Founder's profile URL |
| Outbound email samples | Required | 2–3 recent, names removed |
| Pricing page or slide | If exists | Screenshot or link acceptable |
| Sales brief material | If exists | Any existing positioning docs |
| Call notes or transcripts | If exists | 2–3 recent sales calls |
| Product demo | If exists | Recording or live access |

### Intake Completeness Rule
The kickoff call only proceeds with at minimum: questionnaire + homepage + sales deck + LinkedIn. Everything else enriches the diagnostic but is not a blocker.

If materials are missing, the kickoff call flags this and sets a deadline for delivery before Session 1.

---

## Kickoff Call

**Format:** 30 minutes · Video call
**Timing:** Before Session 1. Ideally 3–5 days before to allow prep time.
**Who:** Founder-CEO + Thorsten

### Agenda

**1. Completeness check (5 min)**
- Review which intake materials were received
- Flag anything missing and agree on delivery deadline
- Confirm nothing has changed since submission (recent pivots, new deals, etc.)

**2. First impressions (10 min)**
- Share one or two high-level observations from initial surface scan
- Not a full diagnostic — enough to signal that analysis has started and to surface any early misalignments
- Example: "Your homepage and your deck seem to be targeting different buyers — we'll dig into this in Session 1"

**3. Logistics (10 min)**
- Confirm session dates for all 3 working sessions (calendar locked now)
- Confirm fast track vs. standard cadence
- Confirm session format (video call, screen share, Notion live)
- Confirm async channel (email / Slack / WhatsApp — founder's preference)

**4. Questions (5 min)**
- Open floor for founder questions about the engagement
- Clarify any ambiguities from intake questionnaire

### Kickoff Output
- Calendar invites sent for Sessions 1–3 + handover
- Notion workspace created (founder invited, not yet populated)
- Any outstanding intake materials flagged with delivery deadline

## Between Kickoff and Session 1: Thorsten's Prep

This is the heaviest prep block. Full surface audit + decision inference + positioning analysis all happen here.

### Prep Work Checklist

**Surface audit:**
- [ ] Homepage audit — 20 buyer questions scored (Q1–Q24, Explicit / Partial / Missing)
- [ ] PULL pattern check: are Q1 (Project), Q9 (Look), Q10 (Lacking) all missing? → flag as structural PULL problem
- [ ] Sales deck audit — 10 buyer questions scored (D1–D10)
- [ ] Outbound email audit — 5 questions scored (O1–O5)
- [ ] LinkedIn profile audit — category, customer, outcome visible?
- [ ] Demo opening audit (if recording available) — 7 questions scored (M1–M7)

**Decision inference:**
- [ ] Map surface gaps to decision gaps using the decision → question mapping table (v5.1 Appendix)
- [ ] Populate decision ledger shell with current status for all 18 core decisions (A/B/C/D types)
- [ ] Identify primary broken lens (Buyability / Travelability / Viability / Reachability / Closability)
- [ ] Draft preliminary X-RAY scorecard

**Positioning analysis:**
- [ ] Competitive landscape scan — who else is in this space, how they position
- [ ] Segment morphology — identify 3–5 plausible ICP configurations
- [ ] Alternative identification — what does the founder's buyer use today instead

**Session 1 content prep:**
- [ ] Diagnostic report draft (Notion page, ready to present)
- [ ] Decision ledger shell populated with current status for each of 18 decisions
- [ ] 2–3 candidate positioning directions sketched (not yet full options — those come after Session 1 alignment)
- [ ] Prepare 3 questions to open the session (situation + symptom + experience)

**Automation note:** Homepage audit, decision inference mapping, and competitive landscape can be 80% automated with AI-assisted analysis. The remaining 20% is judgment on which patterns matter most for this specific founder's stage and market.

## Session 1: Diagnose

**Job:** Find where deals get stuck. Align on the problem before proposing solutions.

**Format:** 90–120 minutes · Video call with screen share
**Notion:** Diagnostic report open and live during session

### Session Structure

**Opening — Situation framing (10 min)**
- 3 targeted questions to surface the founder's current experience:
  - "Walk me through your last three deals — where did each one get stuck?"
  - "When a qualified buyer goes silent, what's the last thing that happened?"
  - "If you could fix one thing about how you explain this product, what would it be?"
- This is not discovery — it's calibration. You've already done the analysis. These questions confirm or challenge the preliminary diagnostic.

**Surface audit walkthrough (40 min)**
- Share screen: pull up the live homepage
- Walk through all 20 buyer questions live (Homepage question set Q1–Q24)
- For each: Explicit / Partial / Missing?
- Run PULL diagnostic: if Q1 (Project), Q9 (Look), and Q10 (Lacking) are all missing → flag as PULL problem, not copy problem — structural reframe required, not wordsmithing
- Score: __/20 explicit. Target green: 16+ explicit
- Repeat for deck (10 questions, D1–D10) and one outbound email (5 questions, O1–O5) if time allows
- Note specific gaps and surface them against the decision ledger

**Decision gap map (20 min)**
- Present the decision ledger with current status for all 18 core decisions, organised by type:
  - **Type A — Positioning (7):** Category, Customer, Alternative, Difference, Outcome, Wedge Use Case, Market Bet
  - **Type B — Trust (4):** Owner/Champion, Proof Type, Effort Shape, Primary Risk
  - **Type C — Movement (4):** First Step, Start Recipe, Risk Reversal, Entry Motion
  - **Type D — Activation (3):** Activation Moment, First Value Path, Time to Value
- Walk through which decisions are fuzzy or missing based on surface evidence
- Name the primary broken lens: Buyability / Travelability / Viability / Reachability / Closability

**Fix-first recommendation (15 min)**
- Present the preliminary fix-first recommendation
- Show the cascade: what unlocks when Zone 1 is fixed
- Align on the positioning bet to develop in Session 2

**Close (10 min)**
- Confirm: "We're going to build 3 positioning options before Session 2. You'll review them before we meet."
- Set implicit observation task: "Between now and Session 2, notice how you explain this in conversations. What phrases seem to land, which ones don't?"
- Preview Session 2 agenda

### Session 1 Output (Thorsten delivers to Notion within 24 hours)
- Finalised buying path diagnostic report (surface audit scores + PULL diagnostic result)
- Homepage scorecard: __/20 explicit, with gaps annotated by decision type
- Decision ledger v1 — all 18 core decisions with current status (A/B/C/D type structure)
- Competitive landscape summary
- Segment morphology draft
- Fix-first recommendation with lens cascade diagram (which lens is red and what it unlocks)

## Between Session 1 and Session 2: Thorsten's Prep

**Purpose:** Translate diagnostic findings into actionable positioning options.

### Prep Work Checklist

**Positioning options:**
- [ ] Develop 3 positioning strategy options based on Session 1 findings
- [ ] For each option: all 7 Type A decisions (Category, Customer, Alternative, Difference, Outcome, Wedge Use Case, Market Bet)
- [ ] Wedge Use Case must be specific enough to visualize Week 1 for the buyer
- [ ] Market Bet must name one prioritised vertical with rationale — no "we serve everyone"
- [ ] Rank options by risk (low / medium / high) with rationale
- [ ] For each option: draft a candidate homepage hero — headline + subheadline, 4 variants
- [ ] Identify the recommended option with explicit reasoning

**Segment work:**
- [ ] Finalise segment morphology (target company + target role, now vs. next)
- [ ] For recommended option: name the wedge use case specifically

**Session 2 content prep:**
- [ ] Positioning options document ready in Notion (visible, scannable, comparable)
- [ ] Homepage hero variations prepared as visual mockups (Figma or inline HTML) — founder should see, not just read

**Founder observation (implicit):**
No formal homework assigned. If founder shares observations about conversations or reactions to framing from Session 1, capture them in the Notion workspace.

## Session 2: Positioning Strategy

**Job:** Lock the positioning bet. Choose the segment, the alternative, the difference, the outcome.

**Format:** 90–120 minutes · Video call with screen share
**Notion:** Positioning options document open and live

### Session Structure

**Opening — Observation debrief (10 min)**
- "Since Session 1 — anything come up in conversations that felt different? Any framing that landed or didn't?"
- Capture any signal. Don't force it if there's nothing. Move on.

**Positioning options walkthrough (35 min)**
- Present 3 positioning options side by side
- For each: explain the bet, the segment it targets, why the alternative creates urgency, what the difference is, what the outcome is
- Show the homepage hero variations for each — visual, not verbal
- Score each option by risk with explicit rationale
- Present the recommendation and the reasoning

**Option selection (20 min)**
- Founder chooses the bet — or chooses a hybrid with clear rationale
- Work through any resistance or uncertainty: "What would need to be true for Option A to be wrong?"
- Lock the primary positioning bet by end of session

**Decisions walkthrough (25 min)**
- Walk through the 7 Type A Positioning decisions for the chosen option
- Lock each one explicitly: Category / Customer / Alternative / Difference / Outcome / Wedge Use Case / Market Bet
- Wedge Use Case and Market Bet are often partially clear after positioning — push for specificity: "Which vertical first, and why?" and "What does Week 1 look like for the buyer?"
- Update decision ledger live in Notion
- Flag any decisions still fuzzy and note what would resolve them

**Close (10 min)**
- Preview Session 3: "We're going to take these 5 locked decisions and build the full messaging spine — all 18 decisions, plus the homepage spine draft"
- Set implicit observation: "If you describe your product to anyone before Session 3, try using [chosen category + alternative framing]. Notice the reaction."

### Session 2 Output (Thorsten delivers to Notion within 24 hours)
- Positioning strategy document (3 options + selected bet with rationale)
- Decision ledger v2 — Type A decisions locked (all 7: Category, Customer, Alternative, Difference, Outcome, Wedge Use Case, Market Bet)
- Segment morphology finalised (target company + role, now vs. next)
- Homepage hero variants for chosen option (Figma)
- Notes on any fuzzy decisions and resolution path

## Between Session 2 and Session 3: Thorsten's Prep

**Purpose:** Expand locked positioning decisions into full messaging spine and homepage spine draft.

### Prep Work Checklist

**Messaging spine:**
- [ ] Expand Type A decisions (already locked) to full 18-decision architecture: lock Types B, C, D
- [ ] Type B — Trust: Owner/Champion, Proof Type, Effort Shape, Primary Risk
- [ ] Type C — Movement: First Step, Start Recipe, Risk Reversal, Entry Motion — ensure Entry Motion is packaged (pilot, diagnostic, entry tier — not just "Request demo")
- [ ] Type D — Activation: Activation Moment, First Value Path, Time to Value
- [ ] For each decision: locked answer in buyer language (outside-in, not internal operations language)
- [ ] Build messaging spine from locked decisions: category statement, problem framing, alternative contrast, difference mechanism, outcome claim, proof direction, first step offer
- [ ] Draft homepage spine: hero + first scroll, all sections mapped to 20 buyer questions

**Messaging repository:**
- [ ] Draft reusable copy blocks by buying stage (Land, Make Sense, Self-Select, Compare, Validate, Commit)
- [ ] Flag which copy blocks are strongest vs. still need founder input

**Session 3 content prep:**
- [ ] Decision ledger v3 — all 18 decisions with proposed answers
- [ ] Homepage spine draft visible in Notion or Figma
- [ ] Messaging repository skeleton populated
- [ ] Prepare: what's still fuzzy and needs founder confirmation in-session

## Session 3: Decisions & Messaging Spine

**Job:** Lock all 18 decisions. Complete the messaging spine. Confirm the homepage spine.

**Format:** 90–120 minutes · Video call with screen share
**Notion:** Decision ledger + homepage spine open and live

### Session Structure

**Opening — Observation debrief (10 min)**
- "Did you try the new framing anywhere this week? What happened?"
- Capture signal. Even informal reactions are diagnostic.

**Decision ledger walkthrough (40 min)**
- Walk through all 18 core decisions by type — confirm each one
- Type A (already locked in Session 2): quick confirmation pass, any refinements?
- Type B — Trust (4): Owner/Champion, Proof Type, Effort Shape, Primary Risk — these often become clear once positioning is locked
- Type C — Movement (4): First Step, Start Recipe, Risk Reversal, Entry Motion — First Step and Entry Motion are critical; Entry Motion should be packaged not vague
- Type D — Activation (3): Activation Moment, First Value Path, Time to Value — require founder input to be specific
- For any still-fuzzy decisions: discuss and lock in-session or mark for Sprint
- Update decision ledger to v3 live in Notion

**Homepage spine review (25 min)**
- Walk through the homepage spine draft section by section
- Founder sees their locked decisions rendered as actual copy
- Revise in-session where needed — Notion is live
- Confirm: does this answer the 18 buyer questions?

**Messaging repository review (10 min)**
- Quick walkthrough of reusable copy blocks
- Flag anything that needs revision before handover

**Handover preview + Full Sprint gap (10 min)**
- Show the gap: "Here's what you now have. Here's what still needs to be built for this to become a full motion." (Homepage fully built, pitch deck, sales brief, demo script, objection map)
- This is not a pitch — it's a diagnosis of what's next
- If Full Sprint conversation is natural, have it. If not, don't force it.

### Session 3 Output (Thorsten delivers to Notion within 48 hours)
- Decision ledger v3 — all 18 core decisions locked (A/B/C/D types complete)
- Messaging spine (complete — built from locked decisions)
- Homepage spine draft (copy-ready, mapped to 20 buyer questions)
- Messaging repository (reusable copy blocks by buying stage: Find, Land, Make Sense, Self-Select, Compare, Validate, Commit)
- Full Sprint gap analysis (what surfaces still need to be built)

## Handover Session

**Format:** 30 minutes · Video call
**Timing:** 2–5 days after Session 3 (gives time to finalise all Notion deliverables)

### Agenda

**Notion workspace walkthrough (20 min)**
- Walk through every page in the Notion workspace
- Confirm founder can navigate and use each deliverable independently
- Confirm Figma assets are linked and accessible
- Answer any questions on individual deliverables

**Usage guidance (5 min)**
- How to use the decision ledger in sales conversations
- How to use the messaging spine for outbound and LinkedIn
- How to use the homepage spine brief when briefing a designer or developer

**Next steps (5 min)**
- If continuing to Full Sprint: confirm dates and transition
- If pausing: agree on check-in point (typically 3–4 weeks)
- Note: Stabilisation Retainer is available if testing support is needed before the Full Sprint

### Handover Deliverables (Complete at Handover)

**Notion workspace (founder has full access):**
- Buying path diagnostic report
- Competitive landscape summary
- Segment morphology
- Positioning strategy document (3 options + selected bet)
- Decision ledger — all 18 decisions locked
- Messaging spine
- Homepage spine draft (copy-ready)
- Messaging repository (copy blocks by buying stage)
- Asset library page (all Figma links)
- Full Sprint gap analysis

**Figma (linked in Notion):**
- Homepage hero variants (from Session 2)
- Homepage spine layout (visual)
- Any surface mockups produced during engagement

## Quality Standards

**Every session must end with:**
- At least one decision locked that was not locked before
- Founder can state the session's output in one sentence
- Next session's agenda is previewed and understood

**Every prep block must produce:**
- At least 2 options for any decision, not just 1 (founder must choose, not just approve)
- A recommendation with explicit reasoning (not "here are options, you decide")
- Notion page updated before next session — founder can read ahead if they want

**Handover is complete when:**
- Founder can navigate the Notion workspace independently
- All 18 decisions are locked with answers in buyer language
- Homepage spine draft is copy-ready (a designer can brief from it)
- Founder knows what's next and why
